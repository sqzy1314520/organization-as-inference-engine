01 · Overview

项目概览

Organization as a Reasoning System（ORS）是一套用于分析和设计人类、AI Agent、数据、工具与组织责任之间协作关系的开放框架。

当信息处理、方案生成和局部执行的成本被 AI 大幅降低后，组织应该如何重新设计信号、意图、任务、能力、决策、责任与反馈之间的关系？
项目状态：概念框架，并非已经经过广泛验证的管理理论。欢迎反例和实验数据。
稳定执行系统 ＋ 动态推理系统 ＝ 更完整的 AI 组织能力
02 · Problem

为什么 AI 越强，组织不一定越快？

信息处理成本↓

搜索、摘要、分类、提取和分析信息。

方案生成成本↓

文本、代码、设计和候选方案批量生成。

局部执行成本↓

标准任务由 Agent 和工作流完成。

但单点变快≠组织变快。瓶颈迁移到：问题定义→上下文→人机组合→方案验证→冲突消解→决策权归属→反馈闭环。

方案生成更快 → 候选更多 → 审核协调增加 → 决策负担上升 → 反馈链路变长 → 组织未必变快
问题不一定是 AI 没创造价值，而可能是组织没有建立吸收新增能力的机制。
03 · Value Traps

四个 AI 价值黑洞

1. 有损传递

原始信号经过标签、摘要、转述和层级汇报后不断损失。

客户反馈 → 标签 → 周报 → 管理层摘要 → 战略结论

2. 瓶颈迁移

AI 消除生成约束后，瓶颈移到评估、验证、决策和协调。

原瓶颈：写不出方案 → 新瓶颈：不知道哪个方案值得执行

3. 协调成本

更多参与方产生重复分析、输出冲突、责任模糊。多 Agent 不会自动产生集体智慧。

4. 反馈断裂

AI 工作流只完成"输入—生成—输出"，结果没有写回系统。

输入→生成→判断→行动→观察→更新→下一轮
04 · Core Model

七层组织推理闭环

未来组织的竞争力不只取决于有多少 AI，而取决于能否让信号、意图、能力、决策、责任与反馈形成低延迟、高保真的闭环。
flowchart TD A["意图 Intent"] --> B["感知 Sense"] B --> C["编译 Compile"] C --> D["路由 Route"] D --> E["探索与执行 Explore/Execute"] E --> F["判断与责任 Judge/Account"] F --> G["反馈与学习 Learn"] G --> A & B & C & D

七层不是七个部门，是组织从信号到行动再到学习必须完成的七种功能。

① 意图

目标、约束、预算、停止条件。

② 感知

信号采集，保留原始证据。

③ 编译

模糊意图→可执行任务结构。

④ 路由

按任务特征组合人/Agent/工具。

⑤ 探索/执行

探索模式并行试错，执行模式标准化输出。

⑥ 判断/责任

评估方案、选择行动、明确责任人。

⑦ 反馈/学习

结果写回上下文、规则、权限和工作流。

编译层 ≠ Prompt Engineering。Prompt 只是接口之一，完整编译还涉及数据、权限、工具、验证和异常处理。
05 · Dual Operating System

执行系统与推理系统

flowchart LR T["任务"] --> Q{"特征"} Q -->|"目标明确/路径成熟"| A["执行系统"] Q -->|"问题模糊/高反馈价值"| B["推理系统"] A --> A1["固定流程·稳定权限·强审计"] B --> B1["动态路由·并行探索·快速验证"] B -->|"方案成熟"| A A -->|"异常增加"| B

执行系统

财务结算、权限管理、合规检查、安全操作。强调稳定、效率、可审计、低方差。

推理系统

市场洞察、产品探索、复杂客户问题。强调感知、假设、验证、反馈与适应。

未知问题 → 多路径探索 → 有效方案 → 标准化 → 固定流程 → 自动化执行
稳定流程 → 异常增加 → 旧规则失效 → 重新定义问题 → 探索新路径
06 · Task Classification

任务分类框架

模糊度风险推荐模式
低低AI 自动执行，人工抽检
低高AI 建议，人工审批
高低AI 并行探索，人类选择
高高人类主导判断，AI 提供分析
正式部署还需加入可逆性、影响范围、证据质量、模型校准和法律责任。
07 · Governance

四种需要重新设计的权力

信息权

谁能查看原始信号、上下文和 Agent 运行记录。最小必要+可回溯。

调用权

谁能调用人、模型、Agent、工具、算力、预算。过窄锁住能力，过宽带来风险。

决策权

AI 可以建议什么、自动执行什么、哪些必须人工审批、谁有否决权。

责任权

谁负责解释、监控、停止、修复并承担后果。"AI 做的"不能成为责任主体。

信息流可以去层级化，责任流不能被彻底模糊化。
08 · Failure Modes

常见退化模式

生成工厂

方案越来多，没人验证和选择。
根因：生成↑，判断未↑。

自动化套壳

原流程不变，中间加 AI，整体周期没缩短。

Agent 会议室

多 Agent 重复冲突输出，缺证据标准和汇总机制。

指标幻觉

指标提升但真实结果变差，系统学会迎合评分。

责任真空

业务、技术、管理者都把错误归因于自动化系统。

永久探索

不断生成新方案，缺预算、收敛标准和执行接口。

09 · Pilot Playbook

最小试点七步法

  1. 选具体任务 — 如"每周分析客户流失原因"，不是"重构客户成功部门"
  2. 画信号到行动链路 — 记录信号来源、摘要次数、问题定义、路由、判断、执行
  3. 识别主瓶颈 — 感知/编译/路由/判断/责任/反馈，哪个环节卡住
  4. 设计最小闭环 — 输入/输出/人机分工/审批节点/失败处理
  5. 设权限与责任 — 谁能访问、调用、审批、停止
  6. 运行有限周期 — 范围、预算、检查比例、终止规则
  7. 前后对比 — 效率/质量/学习/治理四项指标

适合第一批：客户反馈分析内容质检市场情报竞品监测销售线索分级知识问答

不适合第一批：重大法律判断不可逆人事决策核心财务控制医疗诊断

11 · Hypotheses

可证伪假设

H1：反馈延迟

缩短反馈周期比单纯提高生成速度更能改善绩效。

H2：瓶颈迁移

生成成本↓ → 瓶颈移到评估、决策和协调。

H3：编译质量

问题定义和组织上下文的质量不低于模型能力。

H4：双系统

执行+推理双模式优于单一模式。

H5：责任清晰度

明确自动化边界和责任主体提高信任度。

H6：动态路由边界

动态路由有收益的前提：差异+反馈价值>协调成本。
